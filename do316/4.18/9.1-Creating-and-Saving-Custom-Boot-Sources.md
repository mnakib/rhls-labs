
To use custom boot sources, you need to prepare a golden image and make it available to inside an image registry accessible by OpenShift as a container image. OpenShift pulls the image as a Data Source using the `DataImportCron` resource and stores in the `openshift-virtualization-os-images` project, to be used to users to create VMs

The container image that is used to create the Boot Source Disk can be created from a QCOW2 disk or from another container image.



### Create a Boot Source Image from a Container Image

Head to the [Red Hat Ecosystem Catalog](https://catalog.redhat.com/en), filter out to __Containers__ and search for __*rhel9*__. Find the [*rhel9/rhel-guest-image*](https://catalog.redhat.com/en/software/containers/rhel9/rhel-guest-image/6197bdceb4dcabca7fe351d5)

You need to authenticate to the Red Hat registry to be able to pull images from __*redhat.registry.io*__. Create a `service account` in [Registry Service Accounts](https://access.redhat.com/terms-based-registry/) and authenticate to registry.redhat.io using [Podman](https://access.redhat.com/terms-based-registry/token/ocp-virt/docker-login).

```bash
docker login -u=$SERVICE_ACCOUNT_NAME -p=$TOKEN_VALUE registry.redhat.io
```

#### Pull and push the `rhel9/rhel-guest-image:9.7` image

Pull the [`rhel9/rhel-guest-image:9.7`](https://catalog.redhat.com/en/software/containers/rhel9/rhel-guest-image/6197bdceb4dcabca7fe351d5) image from the Red Hat registry, tag the image, then push it to the lab registry

```bash
# Pull the image
podman pull registry.redhat.io/rhel9/rhel-guest-image:9.7

# Tag it
podman tag registry.redhat.io/rhel9/rhel-guest-image:9.7 registry.ocp4.example.com:8443/redhattraining/rhel:9.7

# Login to the lab image registry
podman login -u developer -p developer registry.ocp4.example.com:8443

# Push it to the lab image registry
# Remove signatures because the image is pushed to a private registry
Podman push registry.ocp4.example.com:8443/redhattraining/rhel:9.7 --remove-signatures
```




#### Create the `DataImportCron` resourre

By creating a `DataImportCron`, you are essentially setting up a managed lifecycle for your disk images. This resource will track the external registry and update the `DataSource` whenever a new image version is detected.

#### Create the pull secret used by the `DataImportCron` resouce

First, we need to make the Data Import Cron able to access and pull images from the lab image registry, so we'll need to creata a create a secret in the `openshift-virtualization-os-images` and the `openshift-cnv` project. This secret is used by Data Import Cron for authentication and image pulling against the lab image registry.

```bash
# openshift-virtualization-os-images project containes the Data Sources and data import Crons
oc create secret generic lab-registry-secret --from-literal=accessKeyId=developer --from-literal=secretKey=developer -n openshift-virtualization-os-images

# The openshift-cnv project is where the hyperconverged resource lives
oc create secret generic lab-registry-secret --from-literal=accessKeyId=developer --from-literal=secretKey=developer -n openshift-cnv
```

### Create the `DataImportCron`

```bash
cat > dic_rhel9.7.yaml
```

```yaml
apiVersion: cdi.kubevirt.io/v1beta1
kind: DataImportCron
metadata:
  annotations:
    cdi.kubevirt.io/storage.bind.immediate.requested: "true"
    cdi.kubevirt.io/storage.import.storageClass: ocs-external-storagecluster-ceph-rbd-virtualization
  labels:
    instancetype.kubevirt.io/default-instancetype: u1.medium
    instancetype.kubevirt.io/default-preference: rhel.9
  name: rhel9.7-image-cron
  namespace: openshift-virtualization-os-images 
spec:
  # Defines how many old versions to keep (usually 3 is safe)
  garbageCollect: Outdated
  # The schedule for checking the registry (Mon-Fri, every minute)
  schedule: "* * * * 1-5"
  # Links this cron to a specific DataSource
  managedDataSource: rhel9.7
  
  template:
    spec:
      source:
        registry:
          url: "docker://registry.ocp4.example.com:8443/redhattraining/rhel:9.7"
          secretRef: "lab-registry-secret"
      storage:
        resources:
          requests:
            storage: 10Gi
        storageClassName: "ocs-external-storagecluster-ceph-rbd-virtualization"
```


```bash
# Check that the `DataImportCron` resource has been successfully created
oc get dic -A
```
```bash
NAMESPACE                            NAME                             FORMAT
...

...  
```
```
# Check that the `Datasource` resource has been successfully created
oc get das -A
```
```bash

...

...
```
```bash

```

### Create a Boot Source Image from a QCOW2 Disk


Head to the [Red Hat Customer Portal](https://access.redhat.com/downloads/content/rhel) and the RHEL **Virtualization Images** for **qcow2** disks.


Login to Red Hat registry

mkdir ~/golden-image && cd ~/golden-image

wget "https://access.cdn.redhat.com/content/origin/files/sha256/dc/dc74ad1a9ccd3c62a02cc29f7f4715e47fa0fdaf08a8080dd20906f885f29bae/rhel-10.1-x86_64-kvm.qcow2?user=0ae58ecf7f5bf81a186fae96a66f4b35&_auth_=1773855974_eaa9a3889a01fee6def1c41b49ac52b7" -O rhel-10.1-x86_64-kvm.qcow2


```bash
cat > Containerfile_rhel10.1
```
```Dockerfile
FROM scratch
ADD --chown=107:107 rhel-10.1-x86_64-kvm.qcow2 /disk/
```

Build the image

```bash
podman build -t registry.ocp4.example.com:8443/redhattraining/rhel:10.1 -f Containerfile_rhel10.1
```

This will create a container that will be used in the context of OpenShift Virtualization.


Push it to the classromm image registry

```bash

# Login to the lab image registry
podman login -u developer -p developer registry.ocp4.example.com:8443

# push the image to the lab image registry
podman push registry.ocp4.example.com:8443/redhattraining/rhel:10.1
```
