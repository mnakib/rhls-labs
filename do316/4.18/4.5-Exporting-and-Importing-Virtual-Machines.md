Using the `virtctl`, OpenShift Virtualization can export VMs and their associated disks to import them into another OpenShift Virtualization cluster.


### Create a working project

```bash
oc new-project storage-export-demo
```

### Create a VM and configure its motd file

Create a VM named `httpd-server` inside the project

```bash
oc get vm -n storage-export-demo
```

Login to the VM and update its motd file

```bash
virtctl console httpd-server
```
```bash
cat /etc/motd
```
```bash
sudo su
```
```bash
echo "Authorized Use Only" > /etc/motd
```
```bash
cat /etc/motd
Authorized Use Only
```

Type exit twice and type Press Ctrl+] to exit the console.


### Export the VM

Shut it down first

```bash
virtctl stop httpd-server
```

Verify its status

```bash
oc get vm
```

Check it DV name

```bash
oc get datavolume
```

Export the VM using the `virtctl vmexport download` command

```bash
virtctl vmexport download httpd-server \
  --vm=httpd-server --volume=httpd-server-volume \
  --output=httpd-server-pvc.img.gz
```

### Create the `backup-vm` namespace and import the `httpd-server` VM data volume.

```bash
oc new-project backup-vm
```

Use the `virctl image-upload` command to import the `httpd-server` VM data volume into the `backup-vm` project

```bash
virtctl image-upload dv httpd-server --size=10Gi \
  --storage-class ocs-external-storagecluster-ceph-rbd-virtualization \
  --image-path=httpd-server-pvc.img.gz
```

### Create a VM from the httpd-server VM data volume in the backup-vm project

```bash
virtctl create vm --name=httpd-server-backup \
  --volume-pvc=src:httpd-server | oc apply -f -
```

```bash
oc get vm
```

Login to the console and check the configred motd message appears

<pre>
virtctl console httpd-server-backup
...
httpd-server login: <b>developer</b>
Password: <b>developer</b>
</pre>

<pre>
...
<b>Authorized Use Only</b>
...
</pre>

### Clean up

```bash
oc delete project storage-export-demo

oc delete project backup-vm

rm httpd-server-pvc.img.gz
```
