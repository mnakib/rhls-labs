
How backup is handled depends on the configured data protection model, either CSI Snapshot-based or Kopia Data Mover. Both methods stores metadata in object storage, but the data handling differs depending on the method.


### Verify the backup prerequisites

#### Check the *Bucket* and *BucketClaim* resources


#### Check the *DataProtectionApplication* resource 




#### Check the *BackupStorageLocation*

In the **OpenShift Web Console > Operators > Select the `openshift-adp` project > OADP Operator**

In the **BackupStorageLocation** tab, check that there is *BackupStorageLocation* resource ** available.

```bash
oc get sbl -n openshift-adp
```

### Create the backup resources

In the **Backup** tab, create a *Backup* resource.

- **Name:** backup-vms
- **inludedNamespaces:** <Namespace>-demo
- **exludedResources:** none

Click **Create**.

### Check the backup status

Check the backup status is *Completed*.

Check that the operation was successful by verifying the backup artifacts either through the **OpenShift Web Console** or the `aws` cli command

In **Storage > Object Storage > Buckets > virt-backups-<BucketNumber> > oadp/backups/backup-vm >**, check the backup artifacts.

```bash
aws s3 ls s3:// --endpoint-url https://s3-openshift-storage.apps.ocp4.example.com --no-verify-ssl

aws s3 ls s3://virt-backups-<BucketNumber>/oadp/backups/backup-vms/ --endpoint-url https://s3-openshift-storage.apps.ocp4.example.com --no-verify-ssl
```


Create another backup using 

```yaml
apiVersion: velero.io/v1
kind: Backup
metadata:
  name: backup-vms-bis
  namespace: openshift-adp
spec:
  includedNamespaces:
  - db-demo
  # labelSelector:
    # matchLabels:
      # app: hello-web
```

```bash
oc get deployment velero  -n openshift-adp

alias velero='oc -n openshift-adp exec deployment/velero -c velero -it -- ./velero'

velero get backups

velero describe backup backup-vms \
  --details --insecure-skip-tls-verify
```
