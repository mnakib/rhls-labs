
Use Network Policies to implement segmentation and restrict connections.

OCP supports two types of Network Policies: `NetworkPolicy` and `AdminNetworkPolicy`.

ANP are cluster-wide and have higher priority.

OVN-Kubernetes in OCP implements both types of Network Policies.

Network Policies are additive, so the union of all matching policies determines the allowed traffic

By default, all pods and VMs in a project are accessible from all other pods in or outside the project.

Thus, to secure a namespace, a `default-deny-all` network policy is commonly first created. 


```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-all
  namespace: <NamespaceName>
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress
```

### Create a deny all policy

Create a deny all policy in the `netpol-db-demo` namespace

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-all
  namespace: db-demo
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress
```

```bash
oc get netpol -n db-demo
```

### Create a policy to allow only port 3306 from the `web-demo` from the `web` VM


```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: mariadb-policy
  namespace: db-demo
spec:
  podSelector:
    matchLabels:
      vm.kubevirt.io/name: db
  policyTypes: 
  - Ingress
  - Egress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: web-demo
      podSelector:
        matchLabels:
          vm.kubevirt.io/name: web
    ports:
      - protocol: TCP
        port: 3306
  egress:
    - to:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: openshift-dns
```

```bash
oc get netpol -n db-demo

# Get more info about the policy
oc describe netpol -n db-demo mariadb-policy
```
