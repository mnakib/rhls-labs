# Configuring External Access with Load Balancer Services

### Check the MetalLB Configuration

```bash
# Switch to the metallb-system project.
oc project metallb-system
```

```bash
# List the MetalLB resources in the metallb-system namespace.
# Get the metallb resource
oc get metallb
```

```bash
# Get the metallb ip address pool
oc get ipaddresspool
```

### Create a new working project

```bash
# Create a namespace
oc new-project metallb-demo
```


### Create a MariaDB VM from the `rhel10-mariadb` boot volume



Create a VM named `mariadb-vm` from the `rhel10-mariadb` boot volume in the `metallb-demo` project



Expose the `mariadb-vm` with a service of type Load Balancer

```bash
virtctl expose vm mariadb-vm --name=mariadb \
  --type=LoadBalancer --port=3306 --target-port=3306 --port-name=mysql
```

```bash
# Check the mariadb service was created successfuly
oc get svc
```
```bash
NAME       TYPE           CLUSTER-IP      EXTERNAL-IP     PORT(S)          AGE
...
mariadb    LoadBalancer   172.30.105.97   192.168.50.22   3306:32639/TCP   4s
```

```bash
# Check the service endpoints
oc get endpoints mariadb

# Check the VM IP address
oc get vmi
```


### Check the connectivity to the 'mariadb-vm' server

```bash
nc -zv 192.168.50.22 3306
```

You should see something like this

```
Ncat: Version 7.92 ( https://nmap.org/ncat )
Ncat: Connected to 192.168.50.22:3306.
Ncat: 0 bytes sent, 0 bytes received in 0.02 seconds.
```
