

Velero is a comprehensive Kubernetes backup orchestrator, while Kopia is a high-performance data movement tool. Velero frequently uses Kopia (or Restic) as a backend plugin to handle file-system-level backups of Persistent Volumes (PVs), combining orchestration with efficient, incremental data storage.



### Install the OADP Operator

Install the operator if not already installed. 

**OpenShift Web Console > Operators > OADP Operator > Install**

Check the operator is effectively installed

**OpenShift Web Console > Operators > Select the `openshift-adp` project > OADP Operator**



### Create a S3 bucket claim for the backup location

A S3 Bucket is required to store the backup data. Creating a bucket claim will create a bucket along with the secret and config map needed to access it. The secret contains the ACCESS_KEY and the SECRET_KEY, and the config map contains the bucket name, the S3 endpoint URL, as well as its regional information. Those info are used to connect to bucket by the backup API resource and Data Mover.

To create a bucket in the OpenShift Web console, go to **Storage > Object Storage > Create Bucket**

- **Namespace:** openshift-adp
- **ObjectBackupClaim Name:** virt-backups
- **StorageClass:** openshift-storage.noobaa.io

Inside the `virt-backups` bucket claim page, the **Object Bucket Claim Data** section contains the secret and config map information to connect to the bucket.

Those values can also be seen from the CLI.

```bash
oc get secrets,cm -n openshift-adp

# Extract the secret values
oc extract secret/virt-backups  -n openshift-adp --to=-

# Extract the config map values
oc extract cm/virt-backups  -n openshift-adp --to=-

# Get the route of the storage location
oc get route -n openshift-storage
```

### Create the Data Protection Application API resource

```bash
cat > dpa-resource.yaml
```
```yaml
apiVersion: oadp.openshift.io/v1alpha1
kind: DataProtectionApplication
metadata:
  name: oadp-backup
  namespace: openshift-adp
spec:
  # Using Kopia as the uploader for File System Over Network (FSON)
  configuration:
    velero:
      defaultPlugins:
        - aws
        - openshift
        - csi
        - kubevirt
      uploaderType: kopia
    nodeAgent:
      enable: true
      uploaderType: kopia
      # Enabling Data Mover for CSI snapshots to be moved to object storage
      defaultSnapshotMoveData: true    

  backupLocations:
    - velero:
        config:
          profile: "default"
          region: "us-east-1"
          s3Url: "https://s3-openshift-storage.apps.ocp4.example.com"
          s3ForcePathStyle: "true"
          insecureSkipTLSVerify: "true"
        provider: aws
        # The credential represent the secret in openshift-adp
        # that contains the credentials to connect to the S3 bucket
        credential:
          name: cloud-credentials
          key: cloud
        objectStorage:
          bucket: "virt-backups-<BucketClaimSequentialNumber>"
          prefix: "oadp"
        default: true
```

#### Create the `cloud-credentials` secret

**OpenShift Web Console > Workloads > Secrets > Select the `openshift-adp` project > Create**

- **Secret name:** cloud-credentials
- **Key:** cloud
- **Value:**

```ini
[default]
aws_access_key_id=<ACCESS_KEY_ID_VALUE>
aws_secret_access_key=<SECRET_ACCESS_KEY_VALUE>
```

Create the DPA resource.

```bash
oc apply -f dpa-resource.yaml
```


#### Check the DPA resource health

**OpenShift Web Console > Workloads > Secrets > Select the `openshift-adp` > All Instances >`oadp-backup`**

Check that 

- `oadp-backup` Status Condition is `Reconciled`
- `oadp-backup-1` storage location Status Phase is `Available`


```bash

oc get all -n openshift-adp

```
