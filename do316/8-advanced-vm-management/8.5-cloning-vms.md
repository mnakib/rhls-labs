## Deploy Virtual Machines

To speed up VMs deployment process, you may use on of below methods:

- Install an OS, SysPrep/seal it, turn it off and clone it whenever needed.

- Use templates and then configure the VM with tools such as
cloud-init or Ansible.

## Prepare Virtual Machines for Cloning

For RHEL VMs, use the `virt-sysprep`. The command directly
edits the disk block device to remove all information that is unique to the VM.  

For Microsoft Windows systems, use the `sysprep.exe` command to initiate a _System Out-ofBox-Experience (OOBE)_.

Red Hat OpenShift Virtualization provides the `virtctl guestfs` command to start a container that includes the `virt-sysprep` tool. The container also attaches the VM's PVC as the _/dev/vda_ block device.

The following example shows the execution of the 'virt-sysprep' command to seal the _dsk1_ PVC in the _golden-vms_ namespace:


## Clone Virtual Machines by Using the OpenShift Web Console

OpenShift Virtualization can clone a running VM, but is not supported by Red Hat as this could lead to data inconsistencies.

Stop the VM

```
virtctl stop vm <vm-name>
```

Clone the VM from the Web console
